Splash Page

Technical Description

The splash screen acts as the first point of interaction for users.Some of the key HTML tags I used here are img tag <img> to displays the logo of our website.<h1> and <p> tags Provide a clear headline and mission statement. These are semantically correct for establishing page hierarchy and readability.<ul> tag with <li> tags Used to list team members in aa unordered list. for the Unordered list i used the class <ul class="members"> >/ul to style the list using css. For styling the list I used css ul.members list-style-type: circle; I used <head> and <body>tags. and also I used <alt> tag for all the images for better understanding. In page editor I had to create a seamless transition to homepage after 4s. to do that I used a meta tag <meta http-equiv="refresh" content="4; url=home.html">.

Accessibility

<alt> attribute on the image allows screen readers to describe the logo to visually impaired users.Semantic HTML tags like <h1>, <p>,<ul> and <li> help screen readers interpret the page structure.

Volunteer Page

Technical Description

I used <select> with <option> for sorting volunteer programmes by popularity, rating,cost and duration. Using a dropdown will enhances the interactivity of users with the website.<div class="cards"> and<section class="volunteering-cards"> The card based layout helps in visually grouping each volunteering opportunity. This section Used CSS effects such as "transition: transform 0.3s ease;", scroll-snap-align: start;, hover effect " transform: translateY(-5px); to move the cards up upon pointing the cursor" and to make each card feel interactive. I also used buttons for volunteering cards like "learn more" and "apply" for that I used the tag <button>
<table>inside cards Though tables are generally for tabular data, here it is used creatively to structure program details. It ensures the data is neatly aligned in two columns. <div class="emojirating">A unique feedback tool using plain <span>emoji characters. This is a lightweight and visually intuitive method for collecting emotion-based user feedback, even without JavaScript interactivity. <textarea>for feedback Allows custom, free-form user input. Combined with emoji feedback, it balances structured and open-ended responses.

Accessibility

<label for="sort"> Explicitly connected to the <select> menu via for="sort" and id="sort". This ensures screen readers announce what the dropdown is for. alt attributes on all images Used to improve screen reader output. <small> for review metadata Proper semantic use for less-important info like reviewer names and dates, this helps screen readers de-prioritize extra info. And also I used a placeholder inside the review box so its easy for users to see where to enter the review.

Content Page

Technical Description

The content page includes internal anchor links and a fixed "Go to Top" button. I created another navigation bar to move across the sections in the page and positioned with position: fixed. I used scroll-margin-top and IDs on headings for anchor navigation. Then I used styled images with border-radius,box-shadow and a boarder colour to create a clean layout. I used a navigation bar containing navigation links to different sections of the page. Iused the navigation bar for quick access. for that I used the tag <nav>. and used the class contentnav for style the navigation bar using css. For css styling I used " position : fixed" which fixes the navigation bar to the top of the viewport.Since there is a header for whole website I used margin-top: 70px: Positions the bar 70px from the top to prevent overlapping navigation bar with the header. I used display: flex, justify-content: center, gap: 20px, padding: 0: to arrange the links horizontally with equal spacing and without padding.

Accessibility

All images include alt attributes and all internal links are clearly labelled. Then I used <b> to bold some important facts and information so that it will be easier for the users to point important facts.

Challenges and Lessons Learned

One of the main challenges I faced is when creating the volunteer page I had to make a horizontally scrollable card section.It took me some time to apply the correct codes and css styling to get that job done. I used the CSS "scroll-snap-type: x mandatory;" and "overflow-x: auto" to make that section.
And also it was a challenge to find the relevent pictures to use because there were few royalty free pictures to match our website theme.

Compliance
